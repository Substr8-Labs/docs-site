## Layer 1: Receipts

The atomic primitive. A single execution receipt containing:

- Hash-chained event sequence
- Root hash commitment
- Input/output/environment hashes
- Execution metadata

```json
{
  "run_id": "run-abc123",
  "proof_type": "receipt",
  "root_hash": "sha256:7b2042b7fd1ed84e...",
  "events": [...],
  "status": "completed"
}
```

---

## Layer 2: Signatures

Ed25519 attestation proving who generated the proof:

```json
{
  "signatures": [
    {
      "signer_id": "runtime:81cee45e9ab518db",
      "algorithm": "ed25519",
      "signature": "base64:...",
      "signed_at": "2026-03-17T00:00:00Z"
    }
  ]
}
```

**Verification**: Anyone with the public key can verify.

---

## Layer 3: Proof Graphs

Receipts compose into DAGs representing complex workflows:

```
     Orchestrator (root)
        /    |    \
       в–ј     в–ј     в–ј
   Research Writer Review
       |           в–І
       в–ј           в”Ӯ
    Retry в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ (approval)
```

**Relationship Types:**
- `delegation` вҖ” Parent spawned child
- `retry` вҖ” Same task, new attempt
- `branch` вҖ” Parallel execution
- `approval` вҖ” Human gate passed
- `dependency` вҖ” Waited for result
- `merge` вҖ” Branches joined

## Layer 4: State Proofs

Prove state transitions across runs:

```
State A в”Җв”ҖRunProofв”Җв”Җв–¶ State B в”Җв”ҖRunProofв”Җв”Җв–¶ State C
```

**State Types:**
- `memory` вҖ” Agent memory/context
- `session` вҖ” Conversation state
- `workflow` вҖ” Multi-run workflow state
- `agent` вҖ” Agent definition changes

## Layer 5: Policy Binding

Link runs to governing policies:

```json
{
  "policy_bindings": [
    {
      "policy_type": "acc_token",
      "policy_id": "acc_7f8a9b2c...",
      "policy_hash": "sha256:...",
      "binding_status": "applied"
    }
  ]
}
```

**Binding Status:**
- `applied` вҖ” Policy was in effect
- `violated` вҖ” Policy was violated
- `bypassed` вҖ” Override with audit trail

```
registered вҶ’ active вҶ” paused вҶ’ retired вҶ’ archived
```

**Features:**
- Heartbeat tracking
- Ledger binding
- Run/entry statistics
- Staleness detection

## Protocol Guarantees

| Guarantee | How |
|-----------|-----|
| **Completeness** | Hash chain breaks if events missing |
| **Ordering** | Sequential hashing enforces order |
| **Integrity** | Root hash changes if anything modified |
| **Attribution** | Signatures prove who attested |
| **Provenance** | Proof graphs trace lineage |
| **Settlement** | External anchors provide finality |
